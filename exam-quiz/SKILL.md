# 递进式复习测验系统

**不依赖具体学科。** 从知识点索引和试卷扫描中自动生成考题。轮次不限，每轮根据上一轮薄弱点自适应出题，持续到用户叫停或全部掌握。

## 与其他 skill 的关系

```
learn 编排 → knowledge-graph 构建图谱 → exam-quiz 测验验证
                  │
          知识点索引.md ──→ 题库生成源
          知识关系链.md ──→ 辨析题/推导题生成源
          
          目录中的试卷/习题 ──→ 出题风格参考
```

---

## How to Use

- `/exam-quiz start` — 扫描目录 → 开始第一轮测验
- `/exam-quiz continue` — 继续上次未完成的轮次
- `/exam-quiz review` — 查看当前进度和薄弱点汇总
- `/exam-quiz score` — 显示所有轮次成绩统计
- `/exam-quiz stop` — 结束本轮，保存进度

---

## 设计理念

**不是简单刷题，是自适应的递进复习系统。**

每轮根据上一轮结果自适应调整：薄弱点多出题、已巩固的少出或不出。轮次不限——用户可以一轮接一轮直到满意为止。

轮次递进逻辑：

```
第1轮：全范围扫描 → 标记薄弱点
  ↓
第2轮：集中攻击薄弱点 + 补充遗漏 → 更新薄弱点列表
  ↓
第3轮：跨章节综合 + 顽固点变式 → 标记≥2次错误的顽固点
  ↓
第N轮：只出顽固点，换个角度再考 → 继续直到:
  - 用户说"停"
  - 全部正确率 ≥ 目标（默认90%）
  - 顽固知识点清空
```

每轮正确率 ≥ 70% 可自行决定是否进入下一轮；< 70% 系统建议继续巩固。

## 阶段0：出题素材扫描

### 0.1 知识点来源（必须）

在开始出题前，查找并加载知识点素材：

```
1. 在工作目录及子目录中搜索:
   - 知识点索引.md
   - 公式符号表.md
   
2. 如未找到，尝试搜索上级目录（最多上溯3层）

3. 如仍未找到，询问用户:
   "未找到知识点索引。请提供以下任一:
    (a) 知识点索引文件的路径
    (b) 教材 skill 名称（我将加载后读取）
    (c) 直接告诉我考试范围，我基于范围出题"
```

加载后提取：章节列表、核心概念、关键公式/术语，作为出题范围。

### 0.2 试卷参考扫描（可选，增强出题质量）

在目录中搜索真实试卷/习题作为出题风格参考：

```
扫描目标:
  格式: .pdf .md .txt .docx .jpg .png
  文件名含: 试卷 | 考试 | 期末 | 期中 | exam | test | quiz | 习题 | 练习 | 真题
  搜索范围: 工作目录 + 子目录 + 上级目录(3层)

找到后:
1. 列出找到的文件，让用户确认哪些用作参考
2. 分析试卷:
   - 题型分布
   - 难度层次
   - 高频考点
   - 出题风格（重计算？重概念辨析？重推导？）
   - 分值分布
3. 用分析结果校准出题策略

未找到时:
  → 基于知识点索引直接生成，标注 "📋 未找到试卷参考"
```

### 0.3 用户手动指定参数

```
/exam-quiz start --scope "第1-5章"           # 限定章节
/exam-quiz start --types "单选,填空,判断"    # 限定题型
/exam-quiz start --difficulty "⭐⭐⭐"        # 限定难度
/exam-quiz start --count 20                  # 限定本轮题量
/exam-quiz start --exam "试卷.pdf"           # 指定参考试卷
/exam-quiz start --target 95                 # 目标正确率（默认90%）
/exam-quiz start --record "我的记录.md"       # 指定记录文件路径
```

### 0.4 复习记录文件 — 定位或创建

启动时先判断是否需要跳过历史记录查找：

```
🆕 以下情况跳过搜索，直接创建新记录:

  A. 刚由 learn 编排调用（knowledge-graph 或 book-to-skill 刚刚执行完毕）
     → 素材是全新的，不可能有历史记录，跳过搜索节省时间
     → 直接在工作目录创建 复习记录.md
  
  B. 用户显式表达"从头开始":
     /exam-quiz start --fresh
     → 即使目录中有旧记录也归档，创建新记录
  
  C. 知识点索引的文件修改时间 < 5分钟
     → 很可能是刚生成的，默认创建新记录

📂 以下情况进入搜索流程:

  D. 用户独立调用 /exam-quiz start（非 learn 编排）
  E. /exam-quiz continue（必须找到已有记录）
```

**搜索和创建流程**（仅情况 D/E 执行）：

```
1. 用户通过 --record 参数指定的路径 → 直接使用

2. 搜索工作目录:
   - 查找 复习记录.md / 复习.md / quiz-record.md
   - 搜索子目录及上级目录（最多上溯2层）

3. 找到已有记录:
   → 加载并显示进度摘要:
   
   "📋 找到已有复习记录: d:\...\复习记录.md
    已进行: 2 轮
    总题数: 68 | 总正确率: 73%
    薄弱章节: Ch3(40%), Ch7(50%)
    顽固知识点: 3 个
    
    继续上次进度？(Y/n)"
   
   用户确认后:
   - continue: 加载轮次号、薄弱点、顽固点 → 进下一轮
   - start fresh: 归档旧记录（复习记录_归档_日期.md），创建新记录

4. 未找到已有记录:
   → 在工作目录创建新文件:
   
   "📋 未找到复习记录，将在当前目录创建: d:\...\复习记录.md"
```

**新记录初始化内容**（情况 A/B/C/4）：

```markdown
# 复习记录 — [学科名]

> 开始时间: YYYY-MM-DD HH:MM
> 出题范围: [从知识点索引提取的章节列表]
> 目标正确率: 90%

## 第0轮：出题前基线
| 章节 | 状态 | 备注 |
|------|------|------|
...(列出来源中的所有章节)
```

> `/exam-quiz continue` 始终执行搜索流程。若找不到记录，提示先用 `start`。

## 题库生成原则

### 1. 题型分层（通用）

| 题型 | 用途 | 适用学科 |
|:---|:---|:---|
| 单选题 | 概念辨析、方法选择 | 全部 |
| 多选题 | 易混淆/遗漏的知识点 | 全部 |
| 判断题 | 快速验证理解 | 全部 |
| 填空题 | 关键概念/公式/步骤记忆 | 全部 |
| 简答题 | 概念解释、原理阐述 | 概念型/叙事型 |
| 计算题 | 公式应用、数值求解 | 公式型/混合型 |
| 推导题 | 逐步推导、逻辑链还原 | 公式型/混合型 |
| 改错题 | 常见错误识别与纠正 | 全部 |
| 连线题 | 概念归类、关系匹配 | 概念型/叙事型 |
| 案例分析 | 综合应用、情境分析 | 叙事型 |

每轮根据学科类型和上一轮薄弱点自动调整题型配比。

### 2. 难度标记

| 标记 | 含义 | 首轮分布 | 后续轮次变化 |
|:---|:---|:---|:---|
| ⭐ | 基础：索引中直接可查 | ~25% | 首轮通过后不再出 |
| ⭐⭐ | 理解：需辨析、比较 | ~45% | 薄弱时增加 |
| ⭐⭐⭐ | 综合：跨域、陷阱、推导 | ~30% | 顽固时集中出 |

### 3. 出题依据优先级

```
1. 如果有试卷参考 → 模拟试卷题型/难度/风格
2. 如果有知识点索引 → 以索引中的概念/公式/术语为素材
3. 如果有知识关系链 → 从链中提取辨析题/推导题素材
4. 如果什么素材都没有 → 基于用户描述的范围出题
```

## 递进出题框架

### 第1轮：全范围扫描

**目标**：覆盖全部章节/知识域，建立薄弱点地图

**出题策略**：
- 每个章节均衡出题（通常每章 3-5 题）
- 从基础概念到中等难度
- 题型按学科类型适配
- 有试卷参考时模拟试卷风格

**题量**：覆盖所有章节，通常 30-50 题

**本轮结束后**：
- 统计章节正确率
- 列出薄弱知识点（正确率 < 50%）
- 询问："进入第2轮（集中攻击薄弱点），还是再来一轮同等难度？"

### 第2轮：漏洞补缺

**目标**：强化第1轮薄弱点，补充遗漏

**出题策略**：
- 薄弱点出题量 ×2-3
- 增加辨析题、改错题比例
- 补充首轮未覆盖的细节
- 难度上调到 ⭐⭐-⭐⭐⭐

**题量**：约为首轮的 30-50%（弱点多则多出）

### 第3轮：综合实战

**目标**：跨章节综合应用，识别顽固点

**出题策略**：
- 跨章节综合题（利用知识关系链的跨章连线）
- 推导题（"从A可以推出什么？B可以逆推回A吗？"）
- 改错题（故意在关系链中插入错误："这个推导链哪步错了？"）
- 试卷风格的大题/计算题

**题量**：约首轮的 20-30%

**本轮结束后**：
- 标记顽固知识点（≥2 次错误）
- 顽固点分析表：什么题型？什么错误模式？

### 第N轮（N ≥ 4）：顽固攻克

**目标**：逐个消灭反复出错的知识点

**出题策略**：
- 只出顽固知识点
- 同类题变式重复（换参数、换场景、换问法）
- 要求解释"为什么"而非仅选答案
- 对每个顽固点持续出题直到连续答对 2 次

**终止条件**（任一满足即停止出该知识点的题）：
- 连续答对 2 次 → 标记为 ✅已攻克
- 用户要求跳过 → 标记为 ⏭️ 跳过
- 所有顽固点攻克 → 测验结束 🎉

### 轮次控制

用户可在任意时刻：
- "下一轮" → 进入新一轮
- "停" / "先到这" → 保存进度，随时 `/exam-quiz continue` 继续
- "跳过这个" → 当前知识点不再出题
- "再来一道类似的" → 当前知识点加练一题
- "重来本章" → 指定章节从头出题

## 逐题交互协议

### 出题时

```
**第 N 题** ⭐⭐ 题型 | 章节

题目内容...
```

### 用户答题后

1. **正误判断**：✅ / ❌ / ⚠️（部分对）
2. **正确答案**（如答错）
3. **解析**：为什么对 / 干扰项为什么错 / 关联 `[[知识点]]`

### 自动记录

每题答完后追加到启动时确定的记录文件。写入前确认文件存在，不存在则自动创建。

```markdown
### 第 N 题 ⭐⭐ 题型 — ✅/❌

**题目**：...
**你的答案**：...
**正确答案**：...
**解析**：...
**关联**: [[知识点]]
```

**重要**：先保存上一题，再出下一题。每题立刻落盘，防止中途中断丢失进度。

## 薄弱点追踪

### 记录格式（复习记录文件内）

```markdown
## 第N轮成绩汇总
| 章节 | 题数 | 正确 | 正确率 | 变化 |
|------|------|------|--------|------|
...

## 顽固知识点（≥2次错误）
| 知识点 | 错误次数 | 错误轮次 | 错误题型 | 状态 |
|--------|----------|----------|----------|------|
| xxx    | 3        | 1,2,3    | 单选/填空 | 🔴 |
| yyy    | 2        | 1,3      | 推导      | ✅已攻克 |

## 薄弱点分析
| 薄弱点 | 错误题型 | 可能原因 | 建议 |
|--------|----------|----------|------|
```

### 自适应出题依据

出题前读取记录文件：
1. 已巩固（正确率 > 80% 且 ≥2 次正确）→ 不再出
2. 薄弱（正确率 < 50%）→ 优先出、多出
3. 顽固（≥2 次错误）→ 变式出、反复出
4. 遗漏（索引中有但从未考过）→ 补充出

## 题型模板

### 概念辨析题

素材：知识点索引的相邻行、关系链中 ⇄ ∥ ↔ 的连线。
- "以下哪个是/不是 XXX 的特征？"
- "XXX 和 YYY 的关键区别是什么？"

### 推导题

素材：知识关系链中 → 连线。
- "从 A 可以推导出 ___"（填空）
- "A → B 推导中，关键步骤是？"（简答）
- "以下推导哪步错了？A → B → C"（改错）

### 计算/应用题

素材：公式型知识点索引。
- 给参数求数值
- 公式中某量的单位/意义
- "A 变化 X%，B 如何变？"

### 改错题

在知识点或推理链中故意插入错误：
- 符号错误（正负号、单位）
- 逻辑错误（推导方向反了）
- 概念混淆（A 当成 B）

## Scope & Limits

- 轮次不限，题目质量取决于知识点索引和试卷参考的可用性
- 无知识点索引时需用户描述范围
- 逐题交互、即时反馈、自动记录
- 随时可"暂停"/"继续"/"跳过"
- 记录文件自动搜索或创建：优先级为 --record 参数 > 工作目录已有文件 > 新建

# 知识点图谱构建器

从任意起点（笔记 / PDF 教材 / 已有 skill）出发，构建结构化知识点图谱。自动识别学科类型，适配输出格式。最终产物是 Obsidian 兼容的三层知识网络。

## 两种使用方式

### 独立使用

直接调用，不依赖 learn：

```
/knowledge-graph 笔记目录                        # 纯笔记 → 图谱（无教材核对）
/knowledge-graph 笔记目录 教材.pdf                # 笔记+PDF → 自动调 book-to-skill → 图谱
/knowledge-graph 笔记目录 skill名                 # 笔记+已有 skill → 图谱
/knowledge-graph --extract-only 教材.pdf          # 仅提取教材为 skill
/knowledge-graph 笔记目录 new-skill --merge       # 补充新内容到已有图谱
```

### 被 learn 编排调用

```
learn 在 Step 3b 自动调用 knowledge-graph，传入笔记目录 + 教材 skill slug
→ 无需用户手动指定参数
```

两种方式**完全等价**——learn 只是帮你省掉了手动传参的步骤。

## 核心理念

**原材料不限，输出物统一。** 无论手上是散乱的课堂笔记、PDF 教材、还是现成的 textbook skill——沿着同一条流水线变成三层可导航的知识网络：

```
          ┌─→ 笔记文件夹 ───┐
          │                 ↓
PDF 教材 ─┼─→ [book-to-skill] ─→ 教材 skill ─┐
          │                                   ↓
          已有 skill ─────────────────────→ 知识点图谱引擎
                                              │
                  索引层 ─ "有哪些知识点"（概念/公式/术语按章索引）
                  关系层 ─ "知识点之间怎么关联"（推导/因果/类比/包含）
                  应用层 ─ "遇到问题怎么用"（解题模式/分析框架/实践方法）
```

三层之间通过 Obsidian `[[双向链接]]` 打通，形成可导航的知识网络。

## 学科类型自动识别

在阶段1盘点完成后，根据笔记内容自动判断学科类型，选择对应的输出模板：

| 类型 | 特征 | 索引格式 | 关系链箭头 | 应用模式 | 示例学科 |
|:---|:---|:---|:---|:---|:---|
| **公式型** | 大量 `$...$` 公式、符号密集 | 符号 + 含义 + 单位 | → 推导链为主 | 解题模式 | 物理、化学、工程 |
| **概念型** | 术语定义密集、少公式 | 术语 + 定义 + 章节 | ⊃ 包含 / ⇄ 互释 / ⤳ 因果 | 分析框架 | 生物、医学、法学 |
| **混合型** | 公式 + 概念各半 | 符号表 + 术语表 | → 推导 / ⊃ 分类 / ⇄ 对比 | 解题 + 分析 | 数学、信号处理、经济学 |
| **叙事型** | 时序/事件为主、少公式 | 事件/人物 + 时间 + 意义 | ⤳ 因果 / ⟹ 影响 / ∥ 对比 | 论述框架 | 历史、文学、哲学 |

> 用户也可以手动指定类型：`/knowledge-graph --type formula|concept|mixed|narrative`

## 工作流总览（7个阶段）

| 阶段 | 输入 | 输出 | 核心动作 |
|:---:|:---|:---|:---|
| 0. 原料准备 | PDF/文档 或 skill名 | 教材 skill | 调 book-to-skill 或加载已有 skill |
| 1. 盘点 | 笔记文件夹 | 文件清单 + 学科类型 + 章节缺口 | Glob + 抽样读取 + 类型判断 |
| 2. 概念提取 | 阶段1的内容 | 知识点清单（中间产物） | 按类型适配格式 → 逐文件提取 → 按章归类 |
| 3. 生成笔记 | 阶段2的知识点 + 阶段0的教材 | **Obsidian 知识库文件夹** | 创建章节文件夹 → 逐概念生成 `.md` → 写 frontmatter + 关联 |
| 4. 关系链 | 阶段3的笔记文件 | 补充 `关联` 段 + `知识关系链.md` | 识别跨章关联 → 补充笔记 `[[链接]]` → 生成关系链总览 |
| 5. 一致性 | 阶段0的教材 skill | 修正清单 | 核对术语/约定/缺口 → 修正笔记文件 |
| 6. 应用模式 | 教材 patterns + 笔记 | `应用模式速查.md` | 提取模式 → 关联到笔记 `[[链接]]` |

## 阶段0：原料准备 — 从任意起点到教材 skill

### 0.1 判断入口

```
参数含 .pdf/.epub/.docx 等文档后缀 → 路径A：从原始文档提取
参数含已知 skill 名称              → 路径B：加载已有 skill
参数仅含笔记目录                   → 路径C：跳过阶段0（无教材核对模式）
```

### 0.2 路径A：从原始文档提取（调 book-to-skill）

```
1. 确认文档存在且格式受支持
2. 调用 Skill("book-to-skill")，传入文档路径
3. book-to-skill 自动完成提取并输出到 ~/.claude/skills/<书名-slug>/
4. 记下生成 skill 的 slug，作为后续阶段4的核对基准
```

book-to-skill 不可用或提取失败时，降级为路径C并向用户报告。

### 0.3 路径B：加载已有 skill

```
1. Skill("skill名称") 加载
2. 读取 SKILL.md → Core Frameworks + Chapter Index
3. 读取 glossary.md → 术语表
4. 读取 patterns.md → 解题/应用模式
5. 读取 cheatsheet.md → 公式/常量速查
```

加载失败时提示用户并提供相似名称建议，或降级为路径C。

### 0.4 路径C：无教材核对模式

跳过阶段0和阶段5（一致性核对），只执行阶段1→2→3→4。在 MOC 和速查表中标注 "⚠️ 未经教材核对"。

### 0.5 补充运行（`--merge`）

```
1. 读取已有的章节文件夹 + MOC 索引
2. 加载新教材 skill
3. 对比现有覆盖 vs 教材章节目录
4. 补充缺失章节：新建文件夹 + 生成笔记 + 更新 MOC + 更新速查表
```

## 阶段1：盘点 — 摸清家底 + 判定类型

### 动作

```
1. Glob **/*.md 获取所有 markdown 文件
2. 抽样读取（每个文件前 30 行 + 随机取 2-3 段），不读全文
3. 统计: 文件数、公式密度（$ 出现频率）、术语密度（定义句式频率）
4. 识别章节划分方式（按文件夹 / 按命名前缀 / 按汇总索引文件）
5. 如有汇总索引文件（如"大学物理.md"），优先读取获取章节列表
```

### 输出

```markdown
## 盘点结果

**学科类型**: 公式型（$ 密度 = 高，定义句式 = 中）
**建议输出**: 公式索引 + 推导链 + 解题模式

| 章节 | 文件数 | 知识点数(估) | 状态 |
|:---|:---:|:---:|:---|
| 第1章 | 8 | ~20 | ✅ 完整 |
| 第3章 | 1 | ~3 | ⚠️ 稀疏 |
| ... |

**已覆盖**: 1,2,4,6,7,8,9,10,11,12
**缺失/薄弱**: 3(相对论), 5(流体力学), 13(量子)
```

### 学科类型判定规则

| 判定条件 | 类型 |
|:---|:---|
| `$` 出现频率 > 每10行5次 + 符号-含义-单位结构 | **公式型** |
| 定义句式（"XX是指/定义为"）密集 + 公式稀少 | **概念型** |
| 公式和定义各占相当比例 | **混合型** |
| 含大量时间/地点/人名 + 事件描述多 | **叙事型** |

## 阶段2：概念提取 — 构建知识点清单

> 阶段2 产出的是**中间产物**：一个按章节组织的知识点清单。这份清单将在阶段3 被逐个转化为独立的 `.md` 笔记文件。

### 根据学科类型选择提取格式

#### 公式型 / 混合型：提取符号 + 术语

```
## 第N章 标题

### 公式符号

| 符号 | 含义 | 单位 | 关联笔记 |
|:---:|:---|:---|:---|
| $\vec{F}$ | 力 | $\text{N}$ | [[牛顿第二定律]] |

### 核心术语

| 术语 | 定义 | 关联笔记 |
|:---|:---|:---|
| 惯性系 | 牛顿定律严格成立的参考系 | [[惯性系]] |
```

#### 概念型：知识点索引 = 概念卡片表

```
## 第N章 标题

| 概念 | 定义（一句话） | 层级 | 关联笔记 |
|:---|:---|:---|:---|
| 细胞 | 生命的基本结构和功能单位 | 核心 | [[细胞学说]] |
| 线粒体 | 有氧呼吸的场所，含独立DNA | 细胞器 | [[细胞呼吸]] |
```

#### 叙事型：知识点索引 = 事件/人物表

```
## 第N章 标题

| 事件/人物 | 时间 | 简要 | 意义/影响 | 关联笔记 |
|:---|:---|:---|:---|:---|
| 鸦片战争 | 1840 | 英国对清开战 | 近代史开端 | [[鸦片战争]] |
```

### 通用规则

- 按章节分组，每个知识点记录：名称、定义/公式、来源笔记文件、所属章节
- 同一术语/符号在不同章含义不同的，分章各列一行
- 这份清单是阶段3的输入，**不需要单独输出为文件**（除非用户要求预览）

## 阶段3：生成笔记 — 创建 Obsidian 知识库文件夹

这是知识图谱的**核心产出阶段**。将阶段2 的知识点清单转化为一个完整的、可在 Obsidian 中直接打开的网状笔记库。

### 3.1 创建文件夹结构

在笔记目录下创建以下结构（参考 `D:\资料\课程\Linux\Linux`）：

```
笔记目录/
├── .obsidian/                   ← Obsidian 配置（可选）
├── 00-索引/                     ← MOC 总览 + 学习路径
│   └── [学科名].md              ← 主索引文件（MOC）
├── 01-[第1章名]/                ← 按章节分文件夹
│   ├── 概念A.md
│   ├── 概念B.md
│   └── ...
├── 02-[第2章名]/
│   └── ...
├── ...
├── [学科名]知识点速查.md         ← 快速查阅表（根目录）
├── 知识关系链.md                ← 关系链总览（根目录）
└── (可选) 应用模式速查.md
```

**命名规则**：
- 章节文件夹：`0X-简短英文或中文名`（如 `01-基础概念`、`02-文件管理`），X 为章序号
- 概念笔记文件：**直接用概念名为文件名**，不加序号（如 `管道.md`、`一切皆文件.md`），确保 `[[双向链接]]` 能精确匹配
- 索引文件以学科名命名（如 `Linux Shell 命令和脚本编程.md`）

### 3.2 单个笔记文件格式

**每个知识点生成一个独立的 `.md` 文件**，使用以下统一模板：

```markdown
---
tags: [学科标签, 章节标签, 知识点类型]
aliases: [英文名, 简称1, 简称2]
---

# 知识点名称

## 概述
<1-2 句话概括：这是什么，为什么重要>

## 详细内容
<根据学科类型展开>
<!-- 公式型：公式 + 符号含义 + 推导步骤 -->
<!-- 概念型：定义 + 特征 + 分类/层级 -->
<!-- 混合型：公式 + 概念解释 -->
<!-- 叙事型：时间 + 主体 + 经过 + 意义 -->

## 关联
- [[相关概念A]] — <一句话说明关联性质>
- [[相关概念B]] — <一句话说明关联性质>
<!-- 此段在阶段4 进一步补充 -->

## 书本出处
第X章 (PDF 第Y页) → [[教材.pdf#page=Z|PDF 第Y页]]
```

**强制规则**：
1. **文件名 = 笔记标题 = 知识点名称**，三处一致，确保 `[[链接]]` 精确匹配
2. **frontmatter 必含 `tags` 和 `aliases`**：tags 用于 Obsidian 图谱着色，aliases 用于别名搜索
3. **概述段必写**：每个笔记开头的 1-2 句话，方便 Obsidian 预览（hover 卡片）
4. **关联段预留**：阶段3 先写已在同一章内部明确的关系；跨章关系留到阶段4 补充
5. **书本出处**写到 PDF 页码级别（如有教材 PDF）

### 3.3 主索引文件（MOC）

在 `00-索引/` 下创建以学科名命名的 MOC（Map of Content）：

```markdown
---
tags: [学科标签, MOC, 总览]
aliases: [学科英文名, 简称]
---

# 学科名称

> 作者/来源 · 版本 · 年份

## 全部章节

| 章节 | 标题 | 核心概念 |
|------|------|----------|
| 1 | 标题 | [[概念A]] [[概念B]] [[概念C]] |
| 2 | 标题 | [[概念D]] [[概念E]] |
| ... | ... | ... |

## 核心哲学
<2-5 条贯穿全学科的根本原则，link 到具体笔记>

## 学习路径
<推荐的阅读顺序：先读哪些笔记建立心智模型，再读哪些深入细节>

## 扩展资源
- [[知识点速查]] — 全部公式/命令速查表
- [[知识关系链]] — 跨章关系总览
```

### 3.4 知识点速查表

在根目录创建 `[学科名]知识点速查.md`：

- **公式型**：全部公式符号表（符号 | 含义 | 单位 | 所在笔记）
- **概念型**：全部术语速查（术语 | 定义 | 所在笔记）
- **混合型**：符号表 + 术语表
- **叙事型**：时间轴 + 关键事件速查

每条必须含 `[[双向链接]]` 指回对应的笔记文件。

### 3.5 生成优先级

1. **先建文件夹结构**：一次建好全部章节文件夹
2. **再写主索引 MOC**：确立骨架，决定哪些概念需要独立笔记
3. **逐章生成笔记**：每章内部的概念按依赖顺序生成（先基础后进阶）
4. **最后写速查表**：汇总全部笔记中的符号/术语

### 3.6 合并模式（`--merge`）

如已有 Obsidian 文件夹结构，新增内容时：
1. 新概念 → 在对应章节文件夹下新建 `.md`
2. 已有概念的新信息 → 编辑现有笔记补充
3. 新章节 → 新建章节文件夹 + 更新 MOC 索引表

## 阶段4：关系链 — 补充关联 + 构建关系总览

> 阶段3 生成的每个笔记已有初步的 `## 关联` 段。阶段4 做两件事：(1) 补充跨章节关联；(2) 生成 `知识关系链.md` 总览。

### 4.1 补充笔记内的关联段

遍历阶段3 生成的每个笔记文件，在 `## 关联` 段补充跨章节的 `[[链接]]`：

```
读取 阶段0 的教材 skill → 提取章节间关系
读取 阶段3 的全部笔记 → 识别跨章引用
→ 在每个笔记的 ## 关联 段追加 [[跨章概念]] — 说明
```

### 4.2 知识关系链总览（根目录）

生成根目录的 `知识关系链.md`，作为全部关系的总览文件：

### 核心格式（所有类型通用）

```
[[知识点A]] ⸺关系类型⸺ [[知识点B]] ⸺关系类型⸺ [[知识点C]]

> [!note]- 展开详情
> B 的具体内容（公式/定义/事件描述）
> C 的具体内容

> [!tip]- 关联思路
> A→B→C 的逻辑说明
```

### 关系箭头语义表

| 符号 | 名称 | 含义 | 适用类型 | 例句 |
|:---|:---|:---|:---|:---|
| `→` | 推导 | 数学/逻辑推导，B 可由 A 导出 | 公式/混合 | 牛顿第二定律 → 动量定理 |
| `⇒` | 蕴含 | 逻辑推论，A 成立则 B 成立 | 全部 | 市场失灵 ⇒ 政府干预 |
| `⊃` | 包含 | 分类/组成，A 包含 B | 概念/叙事 | 细胞 ⊃ 细胞核 ⊃ 染色体 |
| `⤳` | 因果 | A 导致 B（时间先后） | 叙事/概念 | 黑船来航 ⤳ 明治维新 |
| `⇄` | 互释 | A 和 B 互为解释/等价 | 全部 | 电场 ⇄ 电势 |
| `∥` | 类比 | A 和 B 结构相似 | 全部 | 平动动能 ∥ 转动动能 |
| `⟹` | 影响 | A 深刻改变了 B | 叙事 | 工业革命 ⟹ 城市化 |
| `↔` | 对比 | A 和 B 相反对立 | 全部 | 保守力 ↔ 非保守力 |

### 设计原则

1. **默认折叠**：`> [!note]-` 末尾 `-` 使 callout 默认折叠。复习时只看到**知识点名称 + 关系箭头**，具体内容被遮住。
2. **跨章串联**：不按章节分，按知识逻辑分。短链合并为长链。
3. **一链一主题**：每条链解决一个问题或讲清一条逻辑线。
4. **多种箭头混用**：一条链内可以 `→` + `∥` + `⇄` 混用，准确表达关系。
5. **展开详情区**含所有节点的具体内容；**关联思路区**用自然语言说明逻辑。

### 不同学科类型的链设计

**公式型**（物理、工程）：
```
[[牛二律]] → [[动量定理]] → [[动量守恒]] ∥ [[角动量守恒]]
```
以推导关系为主，附录加类比对照表。

**概念型**（生物、医学）：
```
[[细胞]] ⊃ [[细胞器]] ⊃ [[线粒体]] ⤳ [[ATP合成]] ⇄ [[有氧呼吸]]
```
以包含关系 + 因果为主线，附录加概念层级树。

**叙事型**（历史、文学）：
```
[[鸦片战争]] ⤳ [[太平天国]] ⟹ [[洋务运动]] ⟹ [[戊戌变法]]
```
以因果 + 影响为主线，附录加时间轴。

**混合型**（数学）：
```
[[极限定义]] → [[导数定义]] → [[微分公式]] ⇄ [[积分公式]]
[[导数]] ∥ [[偏导数]] ⊃ [[梯度]]
```
推导 + 分类混用。

## 阶段5：一致性核对 — 对齐教材

### 核对检查表

| 检查项 | 方法 |
|:---|:---|
| 术语/符号命名 | 教材 glossary vs 笔记文件名 + 标题 |
| 定义表述 | 教材章节 vs 笔记 ## 概述 段 |
| 关系方向 | 教材框架 vs 笔记 ## 关联 段 + 知识关系链.md |
| 正负号/约定 | 教材 cheatsheet vs 笔记中的公式 |
| 缺失章节 | 教材章节目录 vs 已有的章节文件夹 |
| 常量/参数值 | 教材 cheatsheet vs 知识点速查表 |

### 核对流程

```
1. 加载教材 skill
2. 逐项比对，输出修正清单
3. 逐条修正笔记文件（文件名、概述、关联、公式）
4. 补充缺失章节 → 新建章节文件夹 + 生成笔记 + 更新 MOC
```

无教材 skill 时跳过此阶段，在 MOC 和速查表中标注 "⚠️ 未经教材核对"。

## 阶段6：应用模式 — 从「知道」到「会用」

### 模式格式（所有类型通用）

```markdown
### 模式N：模式名称

**适用**: <什么场景>
**步骤**: ①... → ②... → ③...
**关联链**: [[链X：...]]
**注意**: <常见陷阱>

> [!example]- 展开示例
> ...
```

### 不同学科类型的模式侧重

| 类型 | 模式形式 | 示例 |
|:---|:---|:---|
| 公式型 | 解题模式 | 隔离体分析法、守恒律优先策略 |
| 概念型 | 分析框架 | 鉴别诊断流程、法条适用步骤 |
| 混合型 | 解题 + 分析 | 证明策略 + 应用框架 |
| 叙事型 | 论述框架 | 史论结合法、文本分析法 |

### 提取来源

1. 教材 skill 的 `patterns.md`
2. 笔记中的例题/案例思路
3. 用户标注的重点/难点

### 关键规则

每条应用模式必须通过 **关联链** 字段链接到阶段3的某条关系链。使用时从"怎么用"跳转到"为什么"，形成闭环。

---

## 质量规则

1. **文件名 = 标题 = [[链接]] 名** — 三者严格一致，确保 Obsidian 双向链接精确命中
2. **每条笔记独立可读** — 打开任意一个 `.md` 文件，1分钟内能看懂这个概念
3. **关联段必写** — 每个笔记必须有 `## 关联` 段，含至少一个 `[[链接]]`
4. **frontmatter 必含 tags** — 保证 Obsidian 图谱视图可按颜色分类
5. **概述段必写** — 1-2 句话，保证 Obsidian hover 预览卡片有内容
6. **跨章节优先** — 关系链首要目标是打通章节边界
7. **教材是最终仲裁者** — 术语/公式/约定以教材为准
8. **学科类型决定格式** — 不要给概念型学科强加公式块，不要给叙事型学科强加推导链
9. **Obsidian 兼容性优先** — `[[双括号]]` 链接，`$$...$$` / `$...$` 公式，callout 语法严格正确

## 使用示例

### 物理（公式型）：笔记 + 教材 PDF

```
/knowledge-graph d:\资料\课程\物理\大学物理 d:\教材\大学物理学.pdf
```

→ 提取教材 → 盘点（判定公式型） → 提取概念 → **生成 Obsidian 文件夹 + N 个笔记** → 补充关联 → 核对 → 解题模式

### 生物（概念型）：笔记 + 已有 skill

```
/knowledge-graph d:\资料\课程\生物\细胞生物学 cell-bio
```

→ 加载 cell-bio skill → 盘点（判定概念型） → 提取概念 → **生成 Obsidian 文件夹 + N 个笔记** → 补充关联 → 核对 → 分析框架

### 历史（叙事型）：只有笔记

```
/knowledge-graph d:\资料\课程\历史\近代史
```

→ 盘点（判定叙事型） → 提取事件 → **生成 Obsidian 文件夹 + N 个笔记** → 标注 "⚠️ 未经教材核对"

### 手动指定类型

```
/knowledge-graph d:\资料\课程\数学\线性代数 --type mixed
```

### 只提取教材

```
/knowledge-graph --extract-only d:\教材\细胞生物学.pdf
```

# 学习流水线编排器

把"我要学这门课"变成一条自动流水线——从教材提取到笔记整理到图谱构建，最终在 Obsidian 里复习。

## 一句话干什么

```
用户: "帮我学大学物理，笔记在 d:\资料\课程\物理\大学物理，教材是 d:\教材\大学物理学.pdf"

learn → book-to-skill(提取教材) → knowledge-graph(构建图谱) → 完成
```

---

## 三种模式

### 模式 1：完整流水线 `--full`

**触发**: 用户说"我要学XX"并提供了教材 PDF + 笔记目录

```
输入: 教材PDF + 笔记目录
流程: book-to-skill → knowledge-graph
输出: 教材 skill + 知识点索引 + 知识关系链
```

### 模式 2：只整理笔记 `--notes`

**触发**: 用户已有教材 skill 或不需要教材核对，只想整理笔记

```
输入: 笔记目录 [+ 已有 skill 名]
流程: knowledge-graph
输出: 知识点索引 + 知识关系链
```

### 模式 3：只提取教材 `--extract`

**触发**: 用户想先把教材转成 skill，笔记后续再补

```
输入: 教材 PDF
流程: book-to-skill
输出: 教材 skill
```

---

## 执行步骤

### Step 1 — 解析用户意图

从用户消息中提取三个信息（缺哪个就问哪个）：

| 需要知道 | 从哪获取 | 缺了怎么办 |
|:---|:---|:---|
| **学科/课程名** | 用户原话 | 问："你想学什么？" |
| **笔记目录** | 路径参数或上下文 | 问："你的笔记放在哪个文件夹？"或跳过（模式3） |
| **教材来源** | PDF路径 或 skill名 | 问："有教材 PDF 或已有的 skill 吗？"或跳过（模式2） |

### Step 2 — 确定模式并确认

根据 Step 1 收集到的信息，判定模式并向用户确认：

```
📋 学习流水线计划

  学科: 大学物理
  模式: 完整流水线
  
  阶段1: 提取教材
    📖 d:\教材\大学物理学.pdf (474页)
    → 预计 3-5 分钟，生成 college-physics skill
  
  阶段2: 构建知识图谱  
    📝 d:\资料\课程\物理\大学物理 (86 个笔记文件)
    → 生成 知识点索引 + 知识关系链 + 解题模式

  开始？(或说 "跳过提取" / "只整理笔记")
```

### Step 3 — 执行流水线

按模式依次调用子 skill。

**模式1：完整流水线**

```
Step 3a — 提取教材
  Skill("book-to-skill") 传入教材 PDF
  等待完成 → 获取生成的 skill slug → 记作 SKILL_NAME

Step 3b — 构建图谱  
  Skill("knowledge-graph") 传入 笔记目录 + SKILL_NAME
  等待完成 → 获取输出文件列表

Step 3c — 询问自测
  "📝 知识图谱已构建完成。要进行一轮自测吗？"
  
  用户同意 →
    Skill("exam-quiz") 以 fresh 模式启动（图谱刚生成，跳过历史记录搜索）
    exam-quiz 自动基于知识点索引 + 扫描到的试卷出题
    测验按递进轮次进行，用户随时可停
  
  用户拒绝 → 跳到 Step 4

Step 3d — 报告
  ✅ 全部完成
  📖 教材 skill: college-physics
  📝 知识点索引: d:\...\知识点索引.md
  🔗 知识关系链: d:\...\知识关系链.md
  📋 复习记录: d:\...\复习记录.md（如做了测验）
```

**模式2：只整理笔记**

```
Step 3a — 构建图谱
  Skill("knowledge-graph") 传入 笔记目录 [+ skill名(如有)]
  等待完成

Step 3b — 询问自测（同模式1 Step 3c）

Step 3c — 报告
  ✅ 笔记整理完成
  📝 知识点索引: d:\...\知识点索引.md
  🔗 知识关系链: d:\...\知识关系链.md
  ⚠️ 未经教材核对（如未提供教材）
```

**模式3：只提取教材**

```
Step 3a — 提取教材
  Skill("book-to-skill") 传入教材 PDF
  等待完成

Step 3b — 报告
  ✅ 教材提取完成
  📖 教材 skill: ~/.claude/skills/<skill-name>/
  💡 以后可以用 /learn <学科> 笔记目录 <skill-name> 继续整理笔记
```

### Step 4 — 提示下一步

```
💡 后续你可以:
  1. 在 Obsidian 中打开 知识点索引.md + 知识关系链.md
  2. 遮住可折叠区，从概念名称链开始推导
  3. 忘记符号含义时查知识点索引
  4. 随时 /exam-quiz start 开一轮自测
  5. 用 /learn --merge 补充新教材内容
```

## 处理中断与续传

如果某个阶段失败（如 book-to-skill 因 PDF 损坏失败）：

```
❌ 阶段1失败: book-to-skill 无法提取教材 PDF
   原因: PDF 文件已加密/损坏

  选项:
  1. 跳过阶段1，仅整理笔记（无教材核对）
  2. 手动提供教材 PDF 后重试
  3. 提供已有 skill 名称替代
```

如果用户中断后想继续：

```
/learn 大学物理 --resume
```

读取已完成的中间产物，跳过已完成阶段。

## 使用示例

### 完整流水线

```
/learn 大学物理 d:\资料\课程\物理\大学物理 d:\教材\大学物理学.pdf
```

### 快速整理已有笔记（无教材）

```
/learn 线代 d:\资料\课程\数学\线性代数
```

### 笔记 + 已有 skill

```
/learn 细胞生物学 d:\资料\课程\生物\细胞 cell-bio
```

### 只提取教材（为后面做准备）

```
/learn 刑法学 --extract d:\教材\刑法学.pdf
```

### 补充新教材

```
/learn 大学物理 --merge new-physics-skill
```

### 纯自然语言

```
帮我学近代史，笔记在 d:\资料\课程\历史\近代史
```

→ 自动判定模式2（无教材参数），追问是否需要教材核对。

## 与其他 skill 的关系

```
                       ┌──────────────┐
                       │    learn      │
                       │  (编排层·入口) │
                       └───┬────┬──────┘
                           │    │
              ┌────────────┘    └──────────────┐
              ▼                                ▼
    ┌──────────────────┐            ┌──────────────────┐
    │  book-to-skill    │            │ knowledge-graph   │
    │  (原料层·提取)     │ ────────→  │ (加工层·构建)      │
    │  PDF → skill      │   skill    │ 笔记+skill → 图谱  │
    └──────────────────┘            └────────┬─────────┘
                                             │
                                   知识点索引 + 关系链
                                             │
                                             ▼
                                    ┌──────────────────┐
                                    │   exam-quiz       │
                                    │  (检验层·自测)     │
                                    │  图谱 → 递进测验   │
                                    └────────┬─────────┘
                                             │
                                             ▼
                                    ┌──────────────────┐
                                    │    Obsidian       │
                                    │  (消费层·复习)     │
                                    └──────────────────┘
```

learn 本身不处理文档也不构建图谱——它只做两件事：
1. 弄清楚用户想要什么（解析意图）
2. 按正确顺序调用正确 skill（编排流水线）
